## 1️⃣ 전략 네이밍 체계 정리 (일관성 확보)

### 네이밍 원칙

* **코드명(Enum)**: 기술 중심, 내부 구현 기준
* **표시명(UI)**: 사용자 이해 중심, 감성·의사결정 기준
* **AI 전략**은 반드시 `AI_` Prefix 사용 (유료 전용)

---

### 🔓 기본 전략 (Statistical Pick)

> *통계 기반 생성 / 판단·학습 없음*

| 코드명               | 표시 이름      | 성격    | 설명                    |
| ----------------- | ---------- | ----- | --------------------- |
| `BALANCED`        | 균형 추천      | 기본형   | 고빈도와 과거 데이터를 균형있게 조합    |
| `FREQUENT_TOP`    | 자주 나온 번호   | 빈도 중심 | 통계적으로 자주 나온 번호 우선 선택   |
| `OVERDUE_TOP`     | 오래 안 나온 번호 | 역발상   | 최근에 나오지 않은 번호 우선 선택     |
| `WEIGHTED_RANDOM` | 통계 가중 랜덤   | 확률형   | 빈도와 과거 데이터를 결합한 가중치로 추출 |
| `WHEELING_SYSTEM` | 5등 보장 조합   | 시스템   | 5등(3개 일치) 보장 조합 (14게임 권장) |
| `PATTERN_MATCHER` | 패턴 필터링     | 필터형   | 과거 당첨 패턴과 일치하는 조합만 생성   |

**현재 구현 상태**: ✅ 모두 구현 완료 (통계/수학 기반)

---

### 🔒 AI 전략 (AI Reasoned Pick)

> *AI적 사고 구조 기반 / 생성 → 평가 → 제거*

| 코드명                     | 표시 이름        | 핵심 AI 요소 | 구현 상태 |
| ----------------------- | ------------ | -------- | ----- |
| `AI_PATTERN_REASONER`   | AI 패턴 분석 추천  | 패턴 학습    | 🚧 예정 |
| `AI_DECISION_FILTER`    | AI 판단 필터 추천  | 조합 평가    | 🚧 예정 |
| `AI_SIMULATED_LEARNING` | AI 시뮬레이션 추천  | 반복 학습    | 🚧 예정 |
| `AI_WEIGHT_EVOLUTION`   | AI 가중치 진화 추천 | 적응형 판단   | 🚧 예정 |

**구현 계획**: 향후 ML 모델 통합 예정

## 2️⃣ 추가 전략 추천 (AI 유료 전략 중심)

### 🧠 AI_PATTERN_REASONER

> 과거 당첨 패턴과 유사도가 낮은 조합을 AI가 자동 제거

**핵심 기능**:
* 총합 / 홀짝 / 고저 / 연속수 패턴 학습
* 패턴 일치도 60% 이상 조합만 선별
* 통계 기반 → ML 기반 패턴 학습으로 업그레이드

**기술적 차이점** (현재 `PATTERN_MATCHER` vs 향후 `AI_PATTERN_REASONER`):
* **현재**: 통계적 평균/범위 기반 필터링
* **향후**: ML 모델이 복잡한 비선형 패턴 학습

---

### 🧠 AI_DECISION_FILTER

> AI가 "이 조합은 버린다 / 남긴다"를 판단

**핵심 기능**:
* 극단값 조합 자동 제거
* 말이 안 되는 조합 사전 차단
* 분류 모델 기반 이진 판단

**기술적 구현**:
* XGBoost/LightGBM 분류 모델
* 조합 특성 벡터화 → 모델 예측 → 필터링

---

### 🧠 AI_SIMULATED_LEARNING

> AI가 가상의 당첨 상황을 수백~수천 번 시뮬레이션

**핵심 기능**:
* 몬테카를로 시뮬레이션 + 휴리스틱 기반
* 가장 많이 살아남은 조합 선택
* 강화학습 에이전트와 통합 가능

**기술적 구현**:
* 대규모 시뮬레이션 파이프라인
* 보상 함수 기반 조합 평가
* 최적 조합 탐색 알고리즘

---

### 🧠 AI_WEIGHT_EVOLUTION

> AI가 빈도·과거 데이터 가중치를 스스로 조정

**핵심 기능**:
* 패턴 적합도에 따라 가중치 변화
* 고정 공식이 아닌 적응형 전략
* 메타 학습 기반 가중치 최적화

**기술적 구현**:
* 현재 `WEIGHTED_RANDOM`의 고정 50:50 비율을 동적 조정
* 강화학습 또는 베이지안 최적화로 가중치 학습

## 3️⃣ 기본 플랜 vs 유료 플랜 구분

### 🔓 Basic Plan (무료)

**컨셉**

> "통계를 활용한 번호 생성"

**포함 전략**

* ✅ 균형 (`BALANCED`)
* ✅ 자주 나온 번호 (`FREQUENT_TOP`)
* ✅ 오래 안 나온 번호 (`OVERDUE_TOP`)
* ✅ 통계 가중 랜덤 (`WEIGHTED_RANDOM`)
* ✅ 5등 보장 조합 (`WHEELING_SYSTEM`)
* ✅ 패턴 필터링 (`PATTERN_MATCHER`)

**특징**

* 생성 중심
* 판단·학습 없음
* 빠르고 단순
* 통계/수학 기반 알고리즘

**기술적 설명**:
* 모든 전략이 Java로 구현된 통계/수학 알고리즘
* 외부 ML 라이브러리 의존성 없음
* 실시간 계산, 빠른 응답 시간

---

### 🔒 AI Plan (유료)

**컨셉**

> "AI가 번호를 학습·판단·선별"

**포함 전략**

* ✅ 모든 기본 전략 (Basic Plan 포함)
* 🔒 AI 패턴 분석 추천 (`AI_PATTERN_REASONER`)
* 🔒 AI 판단 필터 추천 (`AI_DECISION_FILTER`)
* 🔒 AI 시뮬레이션 추천 (`AI_SIMULATED_LEARNING`)
* 🔒 AI 가중치 진화 추천 (`AI_WEIGHT_EVOLUTION`)

**특징**

* 생성 + 평가 + 선별
* 패턴 기반 평가
* 반복 시뮬레이션
* ML 모델 기반 학습

**AI 요소 설명 (법적 안전 문구)**

> 본 서비스의 AI는 딥러닝 모델이 아닌,
> 과거 데이터 학습·패턴 분석·확률 공간 축소를 수행하는
> **지능형 알고리즘 기반 AI 시스템**입니다.
> 
> AI 전략은 머신러닝 모델(XGBoost, LightGBM 등)을 사용하여
> 복잡한 패턴을 학습하고, 생성된 조합을 평가·선별합니다.

**기술적 설명**:
* ML 모델 서빙 인프라 (Python + Java 통합)
* 특성 엔지니어링 및 모델 학습 파이프라인
* 비동기 처리 및 캐싱으로 응답 시간 최적화

## 6️⃣ 기술적 정직성 보장

### 현재 구현 상태

**Basic Plan 전략**: ✅ 모두 구현 완료
* 모든 전략이 통계/수학 기반 알고리즘
* Java로 구현, 외부 ML 의존성 없음
* 실시간 계산, 빠른 응답

**AI Plan 전략**: 🚧 구현 예정
* ML 모델 통합 필요
* Python ML 서비스와 Java 백엔드 통합
* 모델 학습 및 서빙 인프라 구축 필요

### AI 요소 분류 기준

**AI 요소 있음**:
* 머신러닝 모델 사용 (XGBoost, LightGBM 등)
* 모델 학습 및 예측 과정 포함
* 훈련 데이터를 통한 패턴 학습

**AI 요소 없음**:
* 순수 통계 기반 (빈도, 평균, 분산)
* 수학적 알고리즘 (조합론, 확률론)
* 규칙 기반 필터링

### 법적 안전 문구 (서비스 페이지용)

> ⚠️ **중요 안내**
> 
> 본 서비스는 로또 번호를 **추천**하는 서비스이며,
> 당첨을 **보장**하거나 **예측**하는 서비스가 아닙니다.
> 
> AI 전략은 통계적 패턴 분석을 통해 번호를 선별하지만,
> 로또는 완전한 무작위 추첨이므로 당첨 확률을 높일 수 없습니다.
> 
> 모든 전략은 참고용이며, 최종 선택은 사용자의 판단에 맡깁니다.
