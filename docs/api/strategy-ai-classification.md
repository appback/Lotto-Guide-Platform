## AI 요소 분류 기준

### AI 요소가 있는 전략의 기준

다음 중 하나 이상을 만족하는 전략을 "AI 요소가 있는 전략"으로 분류합니다:

1. **머신러닝 모델 사용**
   - 학습된 모델 (XGBoost, LightGBM, Neural Network 등)
   - 모델 학습 및 예측 과정 포함
   - 훈련 데이터를 통한 패턴 학습

2. **딥러닝 기반**
   - 신경망 모델 사용
   - 복잡한 비선형 패턴 학습

3. **강화학습 기반**
   - 에이전트가 환경과 상호작용하며 학습
   - 보상 함수 기반 최적화

### AI 요소가 없는 전략의 기준

다음과 같은 특성을 가진 전략은 "AI 요소가 없는 전략"으로 분류합니다:

1. **순수 통계 기반**
   - 빈도, 평균, 분산 등 기본 통계량 사용
   - 정렬, 필터링 등 단순 연산

2. **수학적 알고리즘**
   - 조합론, 확률론 기반
   - 결정론적 알고리즘

3. **규칙 기반**
   - 명시적 규칙과 조건문
   - 휴리스틱 기반

## AI 요소가 없는 전략 (통계 기반)

### 1. 고빈도 우선 (FREQUENT_TOP)

**분류**: 통계 기반 정렬 알고리즘

**AI 요소 없음 이유**:
- 단순 빈도 계산 및 정렬
- 학습 과정 없음
- 결정론적 알고리즘

**구현 방식**:
```
1. 각 번호의 빈도(frequency) 계산
2. 빈도 기준 내림차순 정렬
3. 상위 6개 선택
```

**기술 스택**: Java Stream API, 정렬 알고리즘

---

### 2. 과거 데이터 우선 (OVERDUE_TOP)

**분류**: 통계 기반 정렬 알고리즘

**AI 요소 없음 이유**:
- 단순 overdue 값 계산 및 정렬
- 학습 과정 없음
- 결정론적 알고리즘

**구현 방식**:
```
1. 각 번호의 overdue 값 계산 (마지막 등장 이후 경과 횟수)
2. overdue 기준 내림차순 정렬
3. 상위 6개 선택
```

**기술 스택**: Java Stream API, 정렬 알고리즘

---

### 3. 균형 (BALANCED)

**분류**: 제약 조건 기반 랜덤 알고리즘

**AI 요소 없음 이유**:
- 제약 조건 검증 및 랜덤 선택
- 학습 과정 없음
- 규칙 기반 필터링

**구현 방식**:
```
1. 제약 조건 정의 (홀짝비, 합계 범위 등)
2. 제약 조건을 만족하는 조합 생성
3. 최대 1000회 시도하여 유효한 조합 선택
```

**기술 스택**: Java Random, 제약 조건 검증 로직

---

### 4. 추천조합1 (기본 14게임) (WHEELING_SYSTEM)

**분류**: 수학적 조합론 알고리즘

**AI 요소 없음 이유**:
- Round-Robin 방식의 수학적 조합 생성
- 학습 과정 없음
- 결정론적 알고리즘

**구현 방식**:
```
1. 통계적으로 가장 안 나온 9개 번호 제외
2. 36개 번호를 Round-Robin 방식으로 분산
3. 5등 보장을 위한 최소 커버리지 알고리즘
```

**기술 스택**: 조합론, Round-Robin 알고리즘

**수학적 원리**: 
- 36개 번호 중 당첨 번호 6개가 포함되어 있다는 가정
- 각 번호를 여러 조합에 균등하게 분산하여 3개 일치 보장

---

### 5. 가중치 기반 생성 (WEIGHTED_RANDOM)

**분류**: 가중치 기반 확률 분포 알고리즘

**AI 요소 없음 이유**:
- 통계 기반 가중치 계산
- 확률 분포 기반 랜덤 추출
- 학습 과정 없음

**구현 방식**:
```
1. 빈도 가중치 계산 (50%)
2. 과거 데이터(overdue) 가중치 계산 (50%)
3. 가중치 결합 및 정규화
4. 확률 분포에 따라 랜덤 추출
```

**기술 스택**: 
- `WeightedRandomUtil`: 가중치 기반 랜덤 추출 유틸리티
- 확률 분포 생성 및 누적합 기반 선택

**특징**:
- 완전한 무작위가 아닌 통계 기반 확률 분포
- 가중치가 높은 번호일수록 선택 확률이 높음
- 빈도와 과거 데이터를 50:50으로 결합

---

### 6. 패턴 필터링 (PATTERN_MATCHER)

**분류**: 통계 기반 패턴 분석 알고리즘

**AI 요소 없음 이유**:
- 통계적 패턴 분석 (총합, 홀짝비, 고저비, 연속수)
- 학습 과정 없음
- 규칙 기반 필터링

**구현 방식**:
```
1. 과거 당첨 데이터의 패턴 통계 계산
   - 총합 범위 및 평균
   - 홀짝비 범위 및 평균
   - 고저비 범위 및 평균
   - 연속수 포함 비율
2. 생성된 조합의 패턴 분석
3. 과거 패턴과 일치도 계산 (총합 30%, 홀짝비 25%, 고저비 25%, 연속수 20%)
4. 일치도 60% 이상인 조합만 선택
5. 최대 1000회 시도
```

**기술 스택**:
- `PatternAnalyzer`: 패턴 분석 유틸리티
- `WeightedRandomUtil`: 가중치 기반 조합 생성
- 통계적 유사도 계산

**특징**:
- 말이 안 되는 조합을 자동 필터링
- 과거 당첨 패턴과 유사한 조합만 생성
- 다차원 패턴 검증 (총합, 홀짝비, 고저비, 연속수)

**주의사항**:
- AI/ML 모델을 사용하지 않음
- 순수 통계 기반 패턴 분석
- 향후 ML 모델 통합 가능

## 향후 확장 계획

### 예정된 AI 기반 전략

#### 1. 머신러닝 기반 예측 전략 (예정)

**계획된 전략 코드**: `ML_PREDICTOR` (예정)

**구현 계획**:
- XGBoost 또는 LightGBM 모델 사용
- 과거 당첨 데이터를 학습 데이터로 사용
- 특성 엔지니어링:
  - 빈도, overdue, 연속수 등 기존 통계 특성
  - 패턴 특성 (총합, 홀짝비, 고저비)
  - 시계열 특성 (최근 추세)
- 모델 학습 및 예측
- 예측 확률 기반 번호 선택

**기술 스택** (예정):
- Python ML 라이브러리 (XGBoost, LightGBM)
- 모델 서빙 (REST API 또는 Java 통합)
- 특성 추출 파이프라인

**예상 분류**: ✅ AI 요소 있음 (머신러닝 기반)

#### 2. 딥러닝 기반 패턴 학습 전략 (예정)

**계획된 전략 코드**: `DEEP_PATTERN` (예정)

**구현 계획**:
- LSTM 또는 Transformer 모델 사용
- 시계열 데이터로부터 패턴 학습
- 복잡한 비선형 패턴 학습
- 생성된 조합의 패턴 검증

**기술 스택** (예정):
- TensorFlow 또는 PyTorch
- LSTM/Transformer 아키텍처
- 모델 서빙 인프라

**예상 분류**: ✅ AI 요소 있음 (딥러닝 기반)

---

#### 3. 강화학습 기반 최적화 전략 (예정)

**계획된 전략 코드**: `RL_OPTIMIZER` (예정)

**구현 계획**:
- 강화학습 에이전트 (DQN, PPO 등)
- 보상 함수 정의 (당첨 확률, 패턴 일치도 등)
- 환경과의 상호작용을 통한 학습
- 최적 조합 탐색

**기술 스택** (예정):
- RL 라이브러리 (Stable-Baselines3, Ray RLlib)
- 환경 시뮬레이션
- 보상 함수 설계

**예상 분류**: ✅ AI 요소 있음 (강화학습 기반)

## 기술적 세부사항

### 현재 구현 (AI 요소 없음)

**공통 유틸리티**:
- `WeightedRandomUtil`: 가중치 기반 랜덤 추출
- `PatternAnalyzer`: 통계 기반 패턴 분석

**특징**:
- 모든 로직이 Java로 구현
- 외부 ML 라이브러리 의존성 없음
- 빠른 응답 시간 (실시간 계산)
- 결정론적 또는 확률론적 알고리즘

### 향후 구현 (AI 요소 있음)

**필요한 인프라**:
- ML 모델 서빙 인프라
- 모델 학습 파이프라인
- 특성 추출 및 전처리
- 모델 버전 관리

**통합 방식** (예정):
- Python ML 서비스와 Java 백엔드 통합
- REST API 또는 gRPC를 통한 모델 호출
- 비동기 처리 및 캐싱

## 사용자 가이드

### AI 요소가 없는 전략을 선택하는 경우

**장점**:
- 빠른 응답 시간
- 투명한 알고리즘 (해석 가능)
- 안정적인 동작
- 외부 의존성 없음

**적용 시나리오**:
- 실시간 번호 생성이 필요한 경우
- 알고리즘의 동작 방식을 이해하고 싶은 경우
- 통계 기반 접근을 선호하는 경우

### AI 요소가 있는 전략을 선택하는 경우 (향후)

**장점** (예상):
- 복잡한 패턴 학습 가능
- 비선형 관계 탐지
- 예측 정확도 향상 가능성

**단점** (예상):
- 응답 시간 증가 가능
- 모델 해석 어려움 (블랙박스)
- 학습 데이터 의존성
- 인프라 복잡도 증가

**적용 시나리오** (예상):
- 복잡한 패턴 학습이 필요한 경우
- 예측 정확도를 최우선으로 하는 경우
- 충분한 학습 데이터가 있는 경우
